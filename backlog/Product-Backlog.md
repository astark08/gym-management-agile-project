# Product Backlog – Gym Management App

Below is the prioritized backlog with user stories and acceptance criteria.

---

## Priority: High

### 1. Member Check-In System
**User Story:** As a member, I want to check in quickly so I can start my workout without delays.  
**Acceptance Criteria:**
- Member can scan QR or enter ID
- System logs timestamp
- Staff can view check-in history

### 2. Class Scheduling
**User Story:** As a member, I want to book classes so I can reserve my spot.  
**Acceptance Criteria:**
- Members can view class schedule
- Booking respects capacity limits
- Confirmation is displayed

### 3. Trainer Dashboard
**User Story:** As a trainer, I want to see my assigned classes so I can prepare.  
**Acceptance Criteria:**
- Trainer sees upcoming classes
- Trainer can view enrolled members

### 4. Membership Billing
**User Story:** As a gym admin, I want automated billing so payments are consistent.  
**Acceptance Criteria:**
- System processes renewals
- Failed payments generate alerts

---

## Priority: Medium

### 5. Equipment Tracking
**User Story:** As staff, I want to track equipment usage so I know when maintenance is needed.  
**Acceptance Criteria:**
- Equipment list view
- Maintenance log entries

### 6. Admin Dashboard
**User Story:** As an admin, I want to see analytics so I can make decisions.  
**Acceptance Criteria:**
- Displays check-in trends
- Shows class attendance

---

## Priority: Low

### 7. Member Notifications
**User Story:** As a member, I want reminders for classes so I don’t forget.  
**Acceptance Criteria:**
- Sends reminders 1 hour before class

### 8. Trainer Notes
**User Story:** As a trainer, I want to add notes about members.  
**Acceptance Criteria:**
- Notes saved per member
- Only visible to staff
